# 🎨 راهنمای تبدیل MD به DOCX با فرمت‌بندی پیشرفته

## ✨ ویژگی‌های جدید

### 🔤 **فونت‌ها و اندازه‌ها:**
- **عنوان اصلی (H1)**: B Nazanin، 18pt، آبی تیره، Bold
- **عنوان فرعی (H2)**: B Nazanin، 16pt، آبی متوسط، Bold  
- **عنوان سطح ۳ (H3)**: B Nazanin، 14pt، خاکستری، Bold
- **متن عادی**: B Nazanin، 12pt، مشکی
- **نقل قول**: B Nazanin، 11pt، خاکستری تیره، کج
- **کد**: Consolas، 10pt، قهوه‌ای

### 📏 **فاصله‌گذاری:**
- ✅ حفظ خطوط جدید (New Lines)
- ✅ فاصله مناسب بین عناوین  
- ✅ تراز راست برای متن فارسی
- ✅ تورفتگی مناسب برای لیست‌ها و نقل قول‌ها

### 🎯 **فرمت‌های پشتیبانی شده:**

```کد```

## 🚀 استفاده

### **روش ۱: اجرای مستقیم**
```bash
python simple_convert.py
```

### **روش ۲: سفارشی‌سازی فونت**
فایل `font_settings.py` را ویرایش کن:

```python
USER_PREFERENCES = {
    'MAIN_FONT': 'Vazir',        # فونت اصلی
    'CODE_FONT': 'Fira Code',    # فونت کد
    'BASE_SIZE': 14,             # اندازه پایه
    'COLOR_SCHEME': 'colorful',  # طرح رنگی
    'SPACING_MODE': 'loose'      # فاصله‌گذاری
}
```

### **روش ۳: تنظیمات پیشرفته**

برای تغییر رنگ‌ها:
```python
'COLORS': {
    'H1': (0, 51, 102),      # آبی تیره
    'H2': (51, 102, 153),    # آبی متوسط
    'NORMAL': (0, 0, 0),     # مشکی
    # ...
}
```

برای تغییر اندازه‌ها:
```python
'SIZES': {
    'H1': 22,        # عنوان بزرگ‌تر
    'H2': 18,        
    'NORMAL': 12,    
    # ...
}
```

## 🛠️ عیب‌یابی

### مشکل: فونت نمایش داده نمی‌شود
**راه‌حل**: فونت مورد نظر رو نصب کن یا در `font_settings.py` تغییر بده.

### مشکل: فرمت RTL درست نیست
**راه‌حل**: در Word:
1. `File > Options > Advanced`
2. `Show document content` بخش
3. تیک `Use sequence checking` رو بزن

### مشکل: خطوط جدید حفظ نمیشه
**راه‌حل**: اسکریپت جدید این مشکل رو حل کرده! خط‌های خالی در MD به پاراگراف خالی در Word تبدیل میشن.

## 📁 فایل‌های خروجی

فایل‌های DOCX در پوشه `docx_output/` ذخیره میشن با:

- ✅ فرمت‌بندی کامل
- ✅ فونت‌های مناسب  
- ✅ رنگ‌بندی
- ✅ فاصله‌گذاری صحیح
- ✅ پشتیبانی RTL
