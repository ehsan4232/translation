## جریان درخواست

1. کاربران از طریق نام‌های دامنه همچون `api.mysite.com` به وب‌سایت‌ها دسترسی پیدا می‌کنند. معمولاً سیستم نام دامنه (DNS) یک سرویس پولی است که توسط اشخاص ثالث ارائه می‌شود و روی سرورهای ما میزبانی نمی‌شود.

2. نشانی پروتکل اینترنت (IP) به مرورگر یا اپلیکیشن موبایل بازگردانده می‌شود. در این مثال، نشانی IP عبارت است از `15.125.23.214`.

3. پس از دریافت نشانی IP، درخواست‌های پروتکل انتقال ابرمتن (HTTP)[^1] مستقیماً به وب‌سرور شما ارسال می‌شود.

4. وب‌سرور صفحات HTML یا پاسخ JSON را برای نمایش برمی‌گرداند.

## منبع ترافیک

حال بیایید به منبع ترافیک نگاه کنیم. ترافیک به وب‌سرور شما از دو منبع می‌آید: اپلیکیشن وب و اپلیکیشن موبایل.

### اپلیکیشن وب
از ترکیبی از زبان‌های سمت سرور (جاوا، پایتون و غیره) برای مدیریت منطق کسب‌وکار، ذخیره‌سازی و موارد دیگر، و زبان‌های سمت کلاینت (HTML و جاوااسکریپت) برای نمایش استفاده می‌کند.

### اپلیکیشن موبایل
پروتکل HTTP، پروتکل ارتباطی بین اپلیکیشن موبایل و وب‌سرور است. فرمت پاسخ API معمولاً به صورت JavaScript Object Notation (JSON) به دلیل سادگی‌اش استفاده می‌شود.

نمونه‌ای از پاسخ API در فرمت JSON:

```
GET /users/12 – دریافت شیء کاربر برای id = 12
```

---

[^1]: **HTTP:** Hypertext Transfer Protocol (پروتکل انتقال ابرمتن)

**واژه‌نامه:**
- DNS: Domain Name System (سیستم نام دامنه)
- IP: Internet Protocol (پروتکل اینترنت)
- JSON: JavaScript Object Notation
