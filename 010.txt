**ترجمه به فارسی:**

1. کاربران از طریق نام‌دامنه‌ها مانند api.mysite.com به وب‌سایت‌ها دسترسی پیدا می‌کنند. معمولاً سیستم نام‌دامنه (DNS) یک سرویس پولی است که توسط ارائه‌دهندگان شخص ثالث ارائه می‌شود و روی سرورهای ما میزبانی نمی‌شود.  
2. آدرس پروتکل اینترنت (IP) به مرورگر یا اپلیکیشن موبایل بازگردانده می‌شود. در مثال ما، آدرس IP معادل 15.125.23.214 برگشت داده می‌شود.  
3. پس از دریافت آدرس IP، درخواست‌های پروتکل انتقال ابرمتن (HTTP) [1] مستقیماً به وب‌سرور شما ارسال می‌شوند.  
4. وب‌سرور صفحات HTML یا پاسخ‌های JSON را برای نمایش بازمی‌گرداند.  

---  
**حالا بیایید منبع ترافیک را بررسی کنیم.**  
ترافیک ورودی به وب‌سرور شما از دو منبع اصلی تأمین می‌شود:  

• **وب اپلیکیشن**: از ترکیبی از زبان‌های سمت سرور (مانند Java، Python و غیره) برای پردازش منطق کسب‌وکار، ذخیره‌سازی و... استفاده می‌کند و زبان‌های سمت کلاینت (HTML و JavaScript) برای نمایش محتوا به کار می‌روند.  
• **اپلیکیشن موبایل**: پروتکل HTTP، پروتکل ارتباطی بین اپلیکیشن موبایل و وب‌سرور است. فرمت رایج پاسخ API، *نشانه‌گذاری شیء جاوااسکریپت* (JSON) است که به دلیل سادگی برای انتقال داده استفاده می‌شود.  

مثال زیر یک پاسخ API با فرمت JSON را نشان می‌دهد:  
```
GET /users/12 – دریافت شیء کاربر مربوط به شناسه ۱۲
```  

---  
**[1] Hypertext Transfer Protocol (HTTP)**  
*در صورت نیاز به اصطلاح تخصصی اصلی*
