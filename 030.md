4. **کش و بازگشت به کاربر**: CDN تصویر را کش کرده و آن را به کاربر A برمی‌گرداند. تصویر تا زمان انقضای TTL در CDN کش می‌ماند.

## ملاحظات استفاده از CDN

### هزینه

CDNها توسط ارائه‌دهندگان شخص ثالث اداره می‌شوند و شما بابت انتقال داده‌ها به داخل و خارج CDN هزینه پرداخت می‌کنید. 

> **نکتهٔ اقتصادی:** کش کردن فایل‌هایی که به ندرت استفاده می‌شوند مزیت قابل توجهی ندارد، بنابراین بهتر است آن‌ها را از CDN خارج کنید.

### تنظیم زمان انقضای مناسب کش

برای محتوای حساس به زمان، تنظیم زمان انقضای کش اهمیت بالایی دارد:

- **TTL خیلی طولانی**: محتوا ممکن است دیگر تازه نباشد
- **TTL خیلی کوتاه**: باعث بارگیری مکرر محتوا از سرورهای اصلی به CDN می‌شود

### راه‌حل جایگزین برای CDN

باید در نظر بگیرید که وب‌سایت/برنامهٔ شما چگونه با خرابی CDN کنار می‌آید. اگر قطعی موقتی در CDN رخ دهد، کلاینت‌ها باید بتوانند:

1. مشکل را تشخیص دهند
2. منابع را مستقیماً از سرور اصلی درخواست کنند

### ابطال فایل‌ها

می‌توانید یک فایل را قبل از انقضای آن از CDN حذف کنید با انجام یکی از روش‌های زیر:

#### 1. ابطال شیء CDN
استفاده از APIهای ارائه شده توسط فروشندگان CDN

#### 2. نسخه‌بندی شیء
استفاده از نسخه‌بندی شیء برای ارائهٔ نسخهٔ متفاوتی از آن. برای نسخه‌بندی یک شیء، می‌توانید یک پارامتر به URL اضافه کنید:

**مثال:** `image.png?v=2`

در این مثال، شمارهٔ نسخه ۲ به رشتهٔ Query اضافه شده است.

### نتیجهٔ طراحی

شکل ۱-۱۱ طراحی سیستم را پس از اضافه شدن CDN و کش نشان می‌دهد.

---

**واژه‌نامه:**
- CDN: Content Delivery Network (شبکهٔ تحویل محتوا)
- TTL: Time To Live (زمان زنده‌مانی)
- Cache Expiry: انقضای کش
- Origin Servers: سرورهای اصلی
- Invalidate: ابطال
- Object Versioning: نسخه‌بندی شیء
- Query String: رشتهٔ Query
- Cache Hit: برخورد کش
- Cache Miss: عدم برخورد کش
