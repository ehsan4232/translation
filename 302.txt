**مرحله ۲ - ارائه طرح کلی و جلب موافقت**  
طرح به دو بخش اصلی تقسیم می‌شود: انتشار فید و ساخت فید خبری.  

• **انتشار فید:** هنگامی که کاربر پستی را منتشر می‌کند، داده‌های مربوطه در کش و پایگاه داده ذخیره می‌شوند. این پست در فید خبری دوستان او نمایش داده می‌شود.  

• **ساخت فید خبری:** برای سادگی، فرض می‌کنیم فید خبری با جمع‌آوری پست‌های دوستان به ترتیب زمانی معکوس ساخته می‌شود.  

---  

**APIهای فید خبری**  
این APIها روش اصلی ارتباط کلاینت‌ها با سرورها هستند. این APIها مبتنی بر HTTP بوده و به کلاینت‌ها اجازه می‌دهند اقداماتی مانند ارسال پست، دریافت فید خبری، افزودن دوستان و غیره را انجام دهند. در ادامه دو API مهم را بررسی می‌کنیم:  

1. **API انتشار فید**  
   برای انتشار یک پست، یک درخواست POST به سرور ارسال می‌شود. ساختار این API به شرح زیر است:  

   ```
   POST /v1/me/feed
   ```  

   **پارامترها:**  
   • `content`: متن پست.  
   • `auth_token`: برای احراز هویت درخواست‌های API استفاده می‌شود.  

2. **API دریافت فید خبری**  
   ساختار این API برای دریافت فید خبری به صورت زیر است:  

   ```
   GET /v1/me/feed
   ```  

   **پارامترها:**  
   • `auth_token`: برای احراز هویت درخواست‌های API استفاده می‌شود.
