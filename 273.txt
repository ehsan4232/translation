**مرحله ۲ - ارائه طراحی سطح بالا و جلب موافقت**  
این بخش طراحی سطح بالایی را نشان می‌دهد که از انواع مختلف اعلان‌ها پشتیبانی می‌کند:  
• اعلان فشار (Push) آی‌اواس  
• اعلان فشار (Push) اندروید  
• پیامک (SMS)  
• ایمیل  

این بخش به صورت زیر ساختار یافته است:  
• انواع مختلف اعلان‌ها  
• فرآیند جمع‌آوری اطلاعات تماس  
• فرآیند ارسال/دریافت اعلان  

---

**انواع مختلف اعلان‌ها**  
ابتدا به بررسی نحوه عملکرد هر نوع اعلان در سطح بالا می‌پردازیم.  

**اعلان فشار (Push) آی‌اواس**  
برای ارسال اعلان فشار آی‌اواس، عمدتاً به سه جزء نیاز داریم:  

1. **ارائه‌دهنده (Provider)**  
   ارائه‌دهنده درخواست‌های اعلان را ساخته و به سرویس اعلان فشار اپل (APNS) ارسال می‌کند. برای ساخت یک اعلان فشار، ارائه‌دهنده داده‌های زیر را فراهم می‌کند:  
   • **توکن دستگاه (Device Token)**: یک شناسه منحصربه‌فرد برای ارسال اعلان‌های فشار.  
   • **پیلود (Payload)**: یک دیکشنری JSON که محتوای اعلان را در بر می‌گیرد. مثال:  
     ```json
     {
       "aps": {
         "alert": "پیام نمونه",
         "sound": "default"
       }
     }
     ```  

2. **APNS**  
   این یک سرویس راه‌دور ارائه شده توسط اپل است که اعلان‌های فشار را به دستگاه‌های آی‌اواس منتشر می‌کند.  

3. **دستگاه آی‌اواس**  
   این کلاینت نهایی است که اعلان‌های فشار را دریافت می‌کند.  

---

**اعلان فشار (Push) اندروید**  
اندروید از یک فرآیند مشابه برای اعلان‌ها استفاده می‌کند. به جای APNS، معمولاً از **سرویس پیام‌رسانی ابری فایربیس (FCM)** برای ارسال اعلان‌های فشار به دستگاه‌های اندروید استفاده می‌شود.  

*(اصطلاحات تخصصی: APNS = Apple Push Notification Service, FCM = Firebase Cloud Messaging)*
