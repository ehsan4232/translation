## مزایای معماری بدون حالت

در این معماری بدون حالت (stateless)، درخواست‌های HTTP کاربران می‌توانند به هر یک از سرورهای وب ارسال شوند و این سرورها داده‌های حالت را از یک مخزن دادهٔ مشترک بازیابی می‌کنند. داده‌های حالت در یک مخزن دادهٔ مشترک ذخیره می‌شوند و خارج از سرورهای وب نگهداری می‌شوند.

### مزایای کلیدی

یک سیستم بدون حالت دارای ویژگی‌های زیر است:

- **ساده‌تر**: پیکربندی و مدیریت راحت‌تر
- **مقاوم‌تر**: در برابر خرابی‌ها مقاوم‌تر
- **مقیاس‌پذیرتر**: امکان افزودن یا حذف سرورها به راحتی

### طراحی نهایی

**شکل ۱-۱۴** طراحی به‌روز شده با لایهٔ وب بدون حالت را نشان می‌دهد.

---

**واژه‌نامه:**
- Stateless Architecture: معماری بدون حالت
- Shared Data Store: مخزن دادهٔ مشترک
- Stateless Web Tier: لایهٔ وب بدون حالت
- HTTP Requests: درخواست‌های HTTP
- State Data: داده‌های حالت
