\section{دسترسی کاربران به وب‌سایت‌ها از طریق نام دامنه}

کاربران از طریق نام‌های دامنه مانند \lr{api.mysite.com} به وب‌سایت‌ها دسترسی پیدا می‌کنند. معمولاً سرویس \lr{Domain Name System (DNS)}\footnote{سامانه نام دامنه} به‌عنوان یک سرویس پولی توسط ارائه‌دهندگان شخص ثالث ارائه می‌شود و روی سرورهای ما میزبانی نمی‌شود.

\subsection{آدرس \lr{IP} به مرورگر یا برنامه موبایل بازگردانده می‌شود}

در مثال داده‌شده، آدرس \lr{IP} معادل \lr{15.125.23.214} بازگردانده می‌شود.

\subsection{ارسال درخواست‌های \lr{HTTP} به سرور وب}

پس از به‌دست آوردن آدرس \lr{IP}، درخواست‌های \lr{Hypertext Transfer Protocol (HTTP)}\footnote{پروتکل انتقال ابرمتن} مستقیماً به سرور وب شما ارسال می‌شوند.

\subsection{سرور وب صفحات \lr{HTML} یا پاسخ \lr{JSON} را بازمی‌گرداند}

سرور وب صفحات \lr{HTML} یا پاسخ‌های \lr{JSON} را برای نمایش بازمی‌گرداند. در ادامه، به بررسی منبع ترافیک می‌پردازیم.

\section{منبع ترافیک سرور وب}

ترافیک سرور وب شما از دو منبع اصلی تأمین می‌شود:

\begin{itemize}
    \item \textbf{برنامه وب}: از ترکیبی از زبان‌های سمت سرور (مانند \lr{Java}، \lr{Python} و غیره) برای مدیریت منطق کسب‌وکار، ذخیره‌سازی و غیره استفاده می‌کند و از زبان‌های سمت کاربر (مانند \lr{HTML} و \lr{JavaScript}) برای نمایش اطلاعات بهره می‌برد.
    
    \item \textbf{برنامه موبایل}: پروتکل \lr{HTTP} به‌عنوان پروتکل ارتباطی بین برنامه موبایل و سرور وب استفاده می‌شود.
\end{itemize}

\subsection{فرمت پاسخ \lr{API}}

\lr{JavaScript Object Notation (JSON)}\footnote{نشانگر شیء جاوااسکریپت} به‌دلیل سادگی، فرمت رایج برای پاسخ \lr{API} و انتقال داده است. در زیر نمونه‌ای از پاسخ \lr{API} با فرمت \lr{JSON} نشان داده شده است:

\begin{latin}
GET /users/12 – Retrieve user object for id = 12
\end{latin}
