# فصل ۱: مقیاس‌پذیری از صفر تا میلیون‌ها کاربر

طراحی سیستمی که از میلیون‌ها کاربر پشتیبانی کند، چالشی بزرگ است و سفری محسوب می‌شود که نیاز به بازنگری مداوم و بهبود بی‌پایان دارد. در این فصل، سیستمی را می‌سازیم که از یک کاربر پشتیبانی می‌کند و به تدریج آن را مقیاس‌پذیری می‌کنیم تا بتواند به میلیون‌ها کاربر خدمت‌رسانی کند.

پس از مطالعهٔ این فصل، بر تکنیک‌های متعددی تسلط پیدا خواهید کرد که در مواجهه با سؤالات مصاحبهٔ طراحی سیستم به شما کمک خواهد کرد.

## راه‌اندازی یک سرور واحد

> سفر هزار مایلی با یک قدم آغاز می‌شود

ساخت یک سیستم پیچیده نیز فرقی ندارد. برای شروع با چیزی ساده، همه چیز روی یک سرور اجرا می‌شود. شکل ۱-۱ تصویر راه‌اندازی یک سرور واحد را نشان می‌دهد که در آن همه اجزاء روی یک سرور کار می‌کنند: برنامهٔ وب، پایگاه داده، کش و غیره.

برای درک این تنظیمات، بررسی جریان درخواست و منبع ترافیک مفید است. ابتدا بیایید به جریان درخواست نگاهی بیندازیم (شکل ۱-۲).

![تصویر ۹](https://raw.githubusercontent.com/ehsan4232/translation/master/009.png)

## جریان درخواست

۱. کاربران از طریق نام‌های دامنه همچون `api.mysite.com` به وب‌سایت‌ها دسترسی پیدا می‌کنند. معمولاً سیستم نام دامنه¹ یک سرویس پولی است که توسط اشخاص ثالث ارائه می‌شود و روی سرورهای ما میزبانی نمی‌شود.

۲. نشانی پروتکل اینترنت² به مرورگر یا برنامهٔ موبایل بازگردانده می‌شود. در این مثال، نشانی IP عبارت است از `15.125.23.214`.

۳. پس از دریافت نشانی IP، درخواست‌های پروتکل انتقال ابرمتن³ مستقیماً به وب‌سرور شما ارسال می‌شود.

۴. وب‌سرور صفحات HTML یا پاسخ JSON را برای نمایش برمی‌گرداند.

![تصویر ۱۱](https://raw.githubusercontent.com/ehsan4232/translation/master/011.png)

## منبع ترافیک

حال بیایید به منبع ترافیک نگاه کنیم. ترافیک به وب‌سرور شما از دو منبع می‌آید: برنامهٔ وب و برنامهٔ موبایل.

### برنامهٔ وب
از ترکیبی از زبان‌های سمت سرور (جاوا، پایتون و غیره) برای مدیریت منطق کسب‌وکار، ذخیره‌سازی و موارد دیگر، و زبان‌های سمت کلاینت (HTML و جاوااسکریپت) برای نمایش استفاده می‌کند.

### برنامهٔ موبایل
پروتکل HTTP، پروتکل ارتباطی بین برنامهٔ موبایل و وب‌سرور است. فرمت پاسخ API معمولاً به صورت JavaScript Object Notation (JSON) به دلیل سادگی‌اش استفاده می‌شود.

نمونه‌ای از پاسخ API در فرمت JSON:

```
GET /users/12 – دریافت شیء کاربر برای id = 12
```

## پایگاه داده

با افزایش تعداد کاربران، یک سرور کافی نیست و ما به چندین سرور نیاز داریم: یکی برای ترافیک وب/موبایل و دیگری برای پایگاه داده (شکل ۱-۳). جداسازی سرورهای ترافیک وب/موبایل (لایهٔ وب) و پایگاه داده (لایهٔ داده) به آن‌ها این امکان را می‌دهد که به‌طور مستقل مقیاس‌پذیر شوند.

![تصویر ۱۴](https://raw.githubusercontent.com/ehsan4232/translation/master/014.png)

### انتخاب نوع پایگاه داده

شما می‌توانید بین پایگاه دادهٔ سنتی رابطه‌ای و پایگاه دادهٔ غیررابطه‌ای انتخاب کنید. بیایید تفاوت‌های آن‌ها را بررسی کنیم.

#### پایگاه‌داده‌های رابطه‌ای

**پایگاه‌داده‌های رابطه‌ای** همچنین به نام سامانهٔ مدیریت پایگاه دادهٔ رابطه‌ای⁴ یا پایگاه دادهٔ SQL شناخته می‌شوند. محبوب‌ترین‌ها عبارت‌اند از:
- MySQL
- پایگاه دادهٔ اوراکل
- PostgreSQL

**ویژگی‌ها:**
- داده‌ها در جداول و سطرها نمایش داده و ذخیره می‌شوند
- امکان استفاده از عملیات پیوند⁵ بین جداول مختلف با SQL

#### پایگاه‌داده‌های غیررابطه‌ای

**پایگاه‌داده‌های غیررابطه‌ای** نیز به نام پایگاه‌داده‌های NoSQL شناخته می‌شوند. محبوب‌ترین آن‌ها عبارت‌اند از:
- CouchDB
- Neo4j
- Cassandra
- HBase
- Amazon DynamoDB

**انواع پایگاه‌داده‌های غیررابطه‌ای:**
۱. **ذخیره‌سازهای کلید-مقدار**⁶
۲. **ذخیره‌سازهای گراف**⁷
۳. **ذخیره‌سازهای ستونی**⁸
۴. **ذخیره‌سازهای سندی**⁹

> **نکته مهم:** عملیات پیوند به‌طور کلی در پایگاه‌داده‌های غیررابطه‌ای پشتیبانی نمی‌شود.

### کدام یک را انتخاب کنیم؟

برای اغلب توسعه‌دهندگان، پایگاه‌داده‌های رابطه‌ای بهترین گزینه محسوب می‌شوند، زیرا بیش از ۴۰ سال است که وجود داشته و از نظر تاریخی به خوبی عمل کرده‌اند.

با این حال، اگر پایگاه‌داده‌های رابطه‌ای برای موارد استفادهٔ خاص شما مناسب نیستند، ضروری است که گزینه‌های فراتر از پایگاه‌داده‌های رابطه‌ای را در نظر بگیرید.

#### چه زمانی از NoSQL استفاده کنیم؟

پایگاه‌داده‌های غیررابطه‌ای ممکن است انتخاب مناسبی باشند در صورتی که:

- برنامهٔ شما نیاز به تأخیر فوق‌العاده کم دارد
- داده‌های شما ساختار نیافته هستند یا هیچ دادهٔ رابطه‌ای ندارید
- فقط نیاز به سریال‌سازی و دیسریال‌سازی داده‌ها دارید (JSON، XML، YAML و غیره)
- نیاز به ذخیره‌سازی حجم عظیمی از داده دارید
