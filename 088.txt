**ریزسرویس‌های ابری** [4] امروزه بسیار محبوب شده‌اند و معمولاً محدودسازی نرخ (rate limiting) در بخشی به نام **درگاه API** پیاده‌سازی می‌شود. درگاه API یک سرویس کاملاً مدیریت‌شده است که قابلیت‌هایی مانند محدودسازی نرخ، خاتمه SSL، احراز هویت، لیست سفید IP، سرویس‌دهی محتوای ایستا و غیره را پشتیبانی می‌کند.  

در حال حاضر، تنها لازم است بدانید که درگاه API یک **میان‌افزار** است که از محدودسازی نرخ پشتیبانی می‌کند. هنگام طراحی یک محدودکننده نرخ، سوال مهمی که باید از خود بپرسیم این است: **محدودکننده نرخ باید در سمت سرور پیاده‌سازی شود یا در درگاه؟** پاسخ مطلق و قطعی وجود ندارد و این موضوع به عوامل مختلفی مانند **پشته فناوری فعلی شرکت، منابع مهندسی، اولویت‌ها و اهداف** بستگی دارد.  

در ادامه چند راهنمای کلی ارائه می‌شود:  
• **ارزیابی پشته فناوری فعلی** مانند زبان برنامه‌نویسی، سرویس کش و غیره. مطمئن شوید زبان برنامه‌نویسی فعلی شما برای پیاده‌سازی محدودسازی نرخ در سمت سرور کارآمد است.  
• **الگوریتم محدودسازی نرخی را انتخاب کنید** که با نیازهای کسب‌وکار شما سازگار باشد. وقتی همه چیز را در سمت سرور پیاده‌سازی می‌کنید، کنترل کامل بر الگوریتم دارید. اما اگر از یک درگاه API شخص ثالث استفاده کنید، ممکن است گزینه‌های شما محدود شود.  
• اگر از قبل **معماری ریزسرویس** استفاده کرده‌اید و در طراحی خود یک درگاه API برای انجام عملیات احراز هویت، لیست سفید IP و غیره گنجانده‌اید، می‌توانید یک محدودکننده نرخ نیز به آن اضافه کنید.  
• **ساخت سرویس محدودکننده نرخ اختصاصی زمان‌بر است**. اگر منابع مهندسی کافی برای پیاده‌سازی آن را ندارید، استفاده از یک درگاه API تجاری گزینه بهتری است.

**الگوریتم‌های محدودسازی نرخ**  
محدودسازی نرخ را می‌توان با الگوریتم‌های مختلفی پیاده‌سازی کرد که هر کدام مزایا و معایب خاص خود را دارند. اگرچه تمرکز این فصل بر روی الگوریتم‌ها نیست، اما درک کلی از آن‌ها به شما کمک می‌کند الگوریتم مناسب یا ترکیبی از الگوریتم‌ها را برای نیازهای خود انتخاب کنید.  

در ادامه لیستی از الگوریتم‌های محبوب آورده شده است:  
• **سطل توکن (Token bucket)**  
• **سطل نشتی (Leaking bucket)**  
• **شمارنده پنجره ثابت (Fixed window counter)**  
• **ثبت پنجره متحرک (Sliding window log)**  
• **شمارنده پنجره متحرک (Sliding window counter)**
